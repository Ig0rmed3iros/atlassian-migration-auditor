Settings

Application-level configuration for this local Atlassian Audit Platform instance.

Connections are configured per migration

There is nothing to set up here first. Each migration connects its own source and target site with a Jira email and an Atlassian API token (PAT). Credentials are stored encrypted on this machine and never leave it. To reuse a site + token across audits, save it in the Connections vault →

  1. 1
    Generate an API token.

    Visit id.atlassian.com → Security → API tokens and create a token for the account you use on each site.

  2. 2
    Open a migration and connect.

    Paste the site URL, email, and token for the source and the target. The auditor verifies each connection before you continue.

  3. 3
    Run the audit.

    Pick the projects to compare and get a verdict. No global keys to manage.
